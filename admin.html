Panduan Dashboard Admin
Tutorial lengkap semua fitur
Tab Beranda
1
Lihat 4 kartu statistik: Data Klaim (jumlah dokumen), Total Klaim Counter (angka global), Hari Ini, dan Pending.
2
Total Klaim Counter bisa diubah manual — masukkan angka di kotak lalu klik Set.
3
Grafik batang menampilkan klaim 7 hari terakhir secara otomatis.
4
Panel Kesalahan Teknis: aktifkan untuk memblokir semua submit user, isi pesan dulu sebelum aktifkan.
5
Panel Pengumuman: tulis pesan lalu klik Tampilkan → muncul banner kuning di halaman user. Klik Sembunyikan untuk hapus.
6
Toggle EmailJS: matikan jika tidak ingin notifikasi email dikirim saat ada klaim baru.
Tab Daftar Klaim
1
Menampilkan semua username yang berhasil klaim Robux.
2
Gunakan filter tanggal untuk menyaring data berdasarkan hari.
3
Klik ikon salin untuk menyalin username ke clipboard.
4
Tombol hapus (ikon tempat sampah) per baris untuk menghapus satu data.
5
Tombol Hapus Semua untuk menghapus seluruh data klaim — hati-hati, tidak bisa dikembalikan!
6
Catatan: Total Klaim Counter tidak berkurang saat data dihapus.
Tab Permintaan
1
Menampilkan permintaan klaim ulang dari user yang username-nya sudah pernah klaim.
2
Setiap permintaan berisi username, alasan, dan tanggal kirim.
3
Klik Setujui → user akan muncul kembali bisa klaim (tandai approved).
4
Klik Tolak → permintaan ditolak, user tidak bisa klaim ulang.
5
Bisa tambahkan alasan opsional saat setujui atau tolak.
6
Badge merah di sidebar menunjukkan jumlah permintaan pending.
Tab Event
1
Isi form: Robux Didapat, Harga Gamepass, Max Peserta (batas slot).
2
Tanggal & Waktu Berakhir bersifat opsional — kosongkan jika event tidak ada batas waktu.
3
Klik Aktifkan Event → event langsung muncul di halaman user dengan progress bar dan countdown.
4
Klik Matikan Event untuk menghentikan event kapan saja.
5
Tabel Peserta Event menampilkan semua user yang berhasil klaim event.
6
Tombol hapus per baris dan Hapus Semua tersedia untuk mengelola data peserta.
Tab Reset Limit
1
Menampilkan permintaan dari user yang kena rate limit (terlalu banyak percobaan dalam 1 jam).
2
User yang kena limit akan melihat tombol merah "Request Reset Limit" di halaman utama.
3
Setelah user kirim alasan, data masuk ke tab ini dengan status Pending.
4
Klik Setuju → permintaan disetujui. User bisa coba lagi setelah 1 jam berlalu.
5
Klik Tolak → permintaan ditolak.
6
Hapus data yang sudah selesai diproses agar tab tetap rapi.
Kesalahan Teknis (di Beranda)
1
Fitur ini memblokir SEMUA submit dari user secara realtime.
2
Wajib isi pesan sebelum mengaktifkan — pesan tampil di banner merah di halaman user.
3
Setelah aktif, tombol Klaim dan tombol Event di halaman user langsung nonaktif.
4
Gunakan saat sedang maintenance, update, atau ada masalah teknis.
5
Klik Nonaktifkan untuk memulihkan akses user kapan saja.
6
Perubahan bersifat realtime — semua user langsung terpengaruh tanpa perlu refresh.
Tips Penting
  • Selalu logout setelah selesai menggunakan dashboard
  • Jangan bagikan password admin ke siapapun
  • Hapus data klaim secara berkala agar dashboard tetap cepat
  • Aktifkan Kesalahan Teknis dulu sebelum melakukan update besar
  • Counter Total Klaim tidak otomatis berubah saat data dihapus — ini disengaja
